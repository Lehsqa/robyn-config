Що Таке Robyn Framework?

Robyn — це асинхронний Python веб-фреймворк із ядром на Rust, створений для швидкого API-розроблення з мінімальним boilerplate.

  • Швидкодія: Rust runtime та ефективна обробка запитів.
  • Мінімалізм: Простий route-first підхід для REST API.
  • Сучасні можливості: Вбудовані AI, MCP, SSE, GraphQL та інтеграція Rust-коду.
  • Production-ready: Підтримка воркерів, CLI flags і зручного дев-режиму.
from robyn import Robyn

app = Robyn(__file__)

@app.get("/")
def health():
    return {"framework": "Robyn", "status": "ok"}

app.start(port=8080)
Коротко: швидкий Python API framework з Rust-серцем і сучасним DX.

AI та Agents у Robyn

Robyn надає вбудовану інтеграцію з LLM та агентами для прикладних AI-сценаріїв у backend API.

  • AI Configure: Централізоване налаштування провайдера, ключа та моделі.
  • @agent: Декларативне створення агента-асистента з runner-режимом.
  • Agents Examples: Готові приклади запуску агентів з репозиторію Robyn.
from robyn import Robyn, Request
from robyn.ai import agent, configure

configure(
    provider="openai",
    api_key="your-api-key-here",
    model="gpt-4o-mini"
)

@agent(runner="simple", instructions="You are a helpful coding assistant.")
def coding_assistant():
    pass

app = Robyn(__file__)

@app.post("/chat")
async def chat(request: Request):
    body = request.json()
    user_message = body.get("message", "")
    response = await coding_assistant.run(user_message)
    return {"response": response}

MCP: Model Context Protocol

from robyn import Robyn
from robyn.mcp import MCPContext

app = Robyn(__file__)

@app.mcp("greet_user")
def greet_user(name: str, ctx: MCPContext):
    return f"Hello {name}! Your request ID is {ctx.request_id}"

if __name__ == "__main__":
    app.start(port=8080)

Robyn дозволяє вашому додатку виступати як хост для MCP серверів, надаючи агентам доступ до інструментів у стандартний спосіб.

  • Interoperability: Сумісність зі стандартами Anthropic та іншими LLM провайдерами.
  • Tool Discovery: Автоматичне виявлення доступних інструментів та ресурсів.
  • Seamless: Швидке підключення зовнішніх контекстів через CLI або код.

Rust Core & Екстремальна Швидкість

  • Rust Server з коробки: Повноцінний HTTP-сервер, де конфігурація воркерів та потоків виконується прямо в Python коді.
  • Const Requests: Унікальна можливість створювати маршрути, що кешуються на рівні Rust-енергонезалежного шару.
  • Hybrid Code: Можливість підключення власного Rust-коду для критичних обчислень.

Real-time та Сучасні API

from robyn import Robyn, Request, Headers
from robyn.types import HttpMethod
from robyn.robyn import jsonify

app = Robyn(__file__)

@app.get("/sync")
def sync_get(request: Request):
    def my_gen():
        for i in range(10):
            yield f"message {i}\n"
    return my_gen(), Headers({"Content-Type": "text/event-stream"})

@app.get("/client-ip")
def get_client_ip(request: Request):
    return {"ip": request.ip_addr}

@app.route(endpoint="/graphql", methods=[HttpMethod.POST, HttpMethod.GET])
async def graphql_router(request: Request):
    body = request.json()
    query = body.get("query")
    return jsonify({"query": query})

CRUD Порівняння: Robyn vs FastAPI vs Django

Robyn (Minimalistic)

from robyn import Robyn, Request, jsonify

app = Robyn(__file__)
users = {1: {"id": 1, "name": "Anna"}}

@app.get("/user/:id")
def get_u(r: Request):
    uid = int(r.path_params["id"])
    user = users.get(uid)
    return jsonify(user or {"error": "not found"})

@app.post("/user")
def add_u(r: Request):
    data = r.json()
    uid = max(users.keys()) + 1
    users[uid] = {"id": uid, "name": data["name"]}
    return jsonify(users[uid])

@app.delete("/user/:id")
def del_u(r: Request):
    users.pop(int(r.path_params["id"]), None)
    return jsonify({"ok": True})

app.start(port=8080)

FastAPI (Type Safe)

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import uvicorn

app = FastAPI()
users = {1: {"id": 1, "name": "Anna"}}

class UserIn(BaseModel):
    name: str

@app.get("/user/{id}")
def get_u(id: int):
    if id not in users:
        raise HTTPException(status_code=404, detail="not found")
    return users[id]

@app.post("/user")
def add_u(user: UserIn):
    uid = max(users.keys()) + 1
    users[uid] = {"id": uid, "name": user.name}
    return users[uid]

@app.delete("/user/{id}")
def del_u(id: int):
    users.pop(id, None)
    return {"ok": True}

uvicorn.run(app, host="0.0.0.0", port=8000)

Django DRF (Class Based)

from django.urls import path, include
from rest_framework import serializers, viewsets, routers
from .models import User

class UserSer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ["id", "name", "email"]

class UserViewSet(viewsets.ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSer

router = routers.DefaultRouter()
router.register("user", UserViewSet, basename="user")
urlpatterns = [path("api/", include(router.urls))]

# run: python manage.py runserver 0.0.0.0:8000

Що Таке robyn-config?

robyn-config — CLI-надбудова для Robyn, яка автоматизує створення, розширення та підтримку production-ready бекенд-проєктів.

  • create: Генерує основу проєкту (архітектура, Docker, налаштування якості).
  • add: Додає нові сутності та маршрути з авто-реєстрацією.
  • adminpanel: Створює веб-адмінку для CRUD-операцій.
  • DevEx: Менше рутини, більше часу на бізнес-логіку.
# 1) Створення нового проєкту
robyn-config create

# 2) Додавання сутності
robyn-config add

# 3) Генерація адмінпанелі
robyn-config adminpanel

# 4) Запуск застосунку
uv run python -m cli
Результат: структурований Robyn-проєкт із швидким стартом для команди.

robyn-config create

Фундаментальна команда для ініціалізації проєкту. Створює складну архітектуру (DDD/MVC) з усіма налаштуваннями.

  • Архітектура: Підтримка Domain-Driven Design (DDD).
  • Infrastructure: Готова інтеграція з Alembic та SQLAlchemy.
  • Containerization: Docker & Docker Compose з коробки.
